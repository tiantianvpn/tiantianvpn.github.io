第一章 基础设施的隐形博弈

1.1 数据中心的地理迷思

谷歌云遍布全球33个区域和100多个网络边缘节点，但选择不当反而会成为性能杀手。东京区域对东亚用户的理论延迟应低于50ms，但当海底光缆维护时，实际延迟可能暴增300%。2023年第三季度的网络质量报告显示，法兰克福到上海的线路在高峰时段丢包率高达12%，而同期新加坡线路的丢包率仅3.5%。

实战建议：
- 使用Cloud Ping工具实时测试各区域响应时间
- 亚太用户优先选择台湾(asia-east1)或新加坡(asia-southeast1)区域
- 欧美用户考虑比利时(europe-west1)或南卡罗来纳(us-east1)区域

1.2 硬件配置的边际效应

e2-micro实例的共享vCPU在流量激增时会出现明显的性能跳水。测试数据显示，当并发连接数超过200时，1核CPU的响应延迟从80ms陡增至420ms。而将配置升级到n2-standard-2（2vCPU+8GB内存）后，即使并发500连接也能保持150ms以下的稳定响应。

配置黄金法则：
- 基础用户：n1-standard-1（1vCPU/3.75GB内存）
- 中端需求：e2-medium（2vCPU/4GB内存）
- 高性能场景：n2-standard-4（4vCPU/16GB内存）

第二章 网络架构的暗流涌动

2.1 TCP BBR算法的魔法

谷歌云默认的TCP拥塞控制算法可能不适合代理场景。启用BBR算法后，东京到香港的传输吞吐量实测提升47%。通过修改sysctl.conf参数：
bash net.core.default_qdisc = fq net.ipv4.tcp_congestion_control = bbr
可使单连接速度从35Mbps提升至52Mbps。

2.2 DNS解析的蝴蝶效应

使用谷歌云默认DNS可能导致跨境查询绕行。配置/etc/v2ray/config.json使用并行DNS查询：
json "dns": { "servers": [ "https://1.1.1.1/dns-query", "https://8.8.4.4/dns-query", "localhost" ] }
可使DNS解析时间从380ms降至90ms以内。

FAQ

Potatso Lite 是否支持多节点测试？
Potatso Lite 支持手动测试单个节点延迟，但不支持批量自动测速。用户需逐个连接测试节点性能，或者结合其他客户端获取更全面的测速信息。
Clash 策略组如何实现节点优先级切换？
用户可以在 Clash 策略组中设置节点优先级，结合规则文件自动选择延迟低、稳定性高的节点处理流量。优化网页浏览、视频播放和游戏访问速度，提高整体网络效率。
Netch 全局代理与规则代理的区别是什么？
全局代理会将所有流量通过节点转发，适合游戏或全局科学上网；规则代理只针对指定应用或 IP 走节点，优化带宽利用和网络效率，根据不同需求选择使用模式。
Mihomo 是什么？
Mihomo 是 Clash Premium 的开源替代项目，兼容原版 Clash 配置文件并扩展了部分新功能。它性能优异、内存占用低，适用于希望自定义构建代理环境的高级用户。
QuantumultX 的广告屏蔽原理是什么？
QuantumultX 使用分流规则与脚本结合的方式实现广告屏蔽。通过重写 HTTP 请求和阻止特定域名，它能有效去除 App 内广告或网页弹窗，提升浏览体验。
SSR 协议的多路复用功能如何提升性能？
SSR 多路复用允许多个短连接共享同一通道，减少握手次数和加密开销。适合游戏、即时通讯等高频短连接场景，提升网络吞吐量和连接稳定性，提高整体上网体验。
WinXray 的系统代理模式有哪些？
WinXray 支持三种系统代理模式：全局、直连和 PAC 模式。全局模式代理所有流量；直连模式跳过代理；PAC 模式根据规则文件智能判断是否走代理。
如何将 Clash 节点导入到 Sing-box？
Sing-box 支持从 Clash 配置文件中导入节点。用户只需将 YAML 文件放入配置目录或通过命令导入即可。系统会自动解析规则组和节点信息，简化迁移过程。
WinXray 的 PAC 模式适合哪些使用场景？
PAC 模式可根据规则文件动态判断流量是否走代理。适合访问国内外混合网站环境，无需手动切换节点，程序自动分流，提高网络访问效率和稳定性。
Hiddify Next 的多机场订阅如何整合？
Hiddify Next 提供合并订阅功能，可将多个链接导入同一配置，自动去重并分类节点。用户可设置优先级与标签，方便快速选择不同提供商的线路，提升连接灵活性与冗余度。
